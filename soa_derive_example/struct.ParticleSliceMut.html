Use the data argument to request specific fields. This argument can be a reference to any of the marker types in the zip_particle module; or a tuple of reference to types in this module. Get an iterator over multiple fields of this ParticleSliceMut.

Use the data argument to request specific fields. This argument can be a reference or a mutable reference to any of the marker types in the zip_particle module; or a tuple of reference or mutable reference to types in this module. Only fields marked with #[soa_derive(zip)] are available. It is also possible to pass any type implementing IntoIterator after the marker types, and the corresponding values will be forwarded.

This function is only generated when the main struct is public.

Examples

 #[derive(StructOfArray)]
 pub struct Example {
     #[soa_derive(zip)]
     mass: f64,
     #[soa_derive(zip)]
     label: String,
     #[soa_derive(zip)]
     position: [f64; 3],
 }

 use self::zip_example::{Mass, Label, Position};
 let mut vector = ExampleVec::new();

 let mut slice = vector.as_mut_slice();
 for mass in slice.zip_mut(&mut Mass) {
     *mass += 3.0;
 }

 // Mutable position, immutable mass and label
 for (&mass, position, label) in slice.zip_mut((&Mass, &mut Position, &Label)) {
     if label == "foo" {
         position[0] -= mass;
     }
 }
